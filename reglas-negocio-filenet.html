4.2 Campos condicionales por Tipo Movimiento

Si MN04 (Modificación): Aparece campo "Tipo de Modificación" (obligatorio)

Si MN12 (Error Títulos): Aparece campo con 10 opciones de corrección

Si MN13 (Servicios Capitalización): Aparece "Tipo de Servicio" → si Servicios técnicos: 18 opciones (SC01-SC18)

4.3 Conocimiento del Cliente (ContinuarProceso)

El sistema consulta automáticamente el Servicio de Marcas para verificar AML/SARLAFT.

Campo ContinuarProceso tiene 2 posiciones:

  • Pos 1: Tomador requiere análisis (S/N)
  • Pos 2: Asegurado requiere análisis (S/N)

Efecto: Si = "S" → se oculta campo "Actualizar Estado" en escritorios: Análisis Riesgo, CAV, Conocimiento, Análisis Seguro

Regla especial: Si Tomador = Asegurado o Ramo 801 (Capitalización) → mismo resultado para ambos

4.4 Exceptuado y Excepciones

Exceptuado

S = Puede saltar pasos de análisis
N = Flujo normal completo

Excepción de Negocio

true = Excepción aprobada nivel superior
false = Flujo estándar

Tipo Excepción (solo Autos)

  • ATC — Convertibles
  • ACB — Blindados
  • ACP — Placa pública uso particular
  • APF — Modelo >15 años
  • MFP — Marcas fuera de políticas
  • MNJ — Motocicletas
  • TSP — Taxis servicio público
  • VPE — Vehículos pesados/carga
  • VSV — Supera valor asegurado

4.5 Tomador = Asegurado

TomadorAsegurado = "SI" → Solo info tomador
TomadorAsegurado = "NO" → Campos adicionales asegurado (tipo doc, número, nombres, apellidos)

8. Ciclo de Vida del Caso

8.1 Cómo se crea un caso

Un sistema externo (Portal Intermediarios, Simón Servicio, Gestor Correos) envía a Filenet:

Dato requeridoEjemplo
Nombre del flujoSuscripción y Emisión
Ramo / Producto / SubproductoVida / Individual / Pesos
Tipo Movimiento NegocioMN02 (Nuevo negocio)
Datos tomadorTipo doc, número, nombres, apellidos
Datos asegurado (si difiere)Tipo doc, número, nombres, apellidos
Clave asesor comercial5 dígitos (10000-99999)
Localidad radicaciónBogotá
Director comercial—
Documentos adjuntosPDFs, imágenes
Observaciones inicialesMáx 512 caracteres

Al crearse automáticamente:

  1. Se asigna consecutivo único (ej: 214527)
  2. Se establece fecha/hora de creación
  3. Se registra sistema de origen
  4. Se consulta Servicio de Marcas (conocimiento cliente)
  5. Caso llega al primer escritorio (Análisis de Riesgo)
  6. Estado se establece como "Pendiente"

8.2 Cómo se gestiona un caso en un escritorio

PasoAcciónDetalle
1Abrir casoClick "Open" desde bandeja de trabajo
2Revisar informaciónDatos caso: ramo, producto, movimiento, cliente
3Revisar documentosPestaña "Attachments" → PDFs adjuntos
4Consultar historialPestaña "History" → acciones previas, quién, cuándo
5Completar camposActualizar Estado Negocio + campos obligatorios
6Escribir comentarioObservación explicando decisión (máx 512 chars)
7DespacharPresionar botón según decisión → caso va al destino

Alternativas: "Save" = guardar sin despachar | "Cancel" = cerrar sin guardar

8.3 Qué pasa al presionar "Completar"

Escritorios de decisión

(Análisis Riesgo, Jefes Técnicos, Líderes Operaciones, etc.)

  • "Completar" = gestión en ese escritorio terminó exitosamente
  • Motor Filenet decide automáticamente el siguiente paso
  • Caso puede ir a otro escritorio o finalizarse

Escritorios paso único

(CAV, Outsourcing, Gestión Documental, Conocimiento)

  • "Complete" es la única opción
  • Caso siempre avanza al siguiente paso definido
  • Usuario no controla destino

Cuando el caso termina

  • Se retira del flujo de trabajo
  • No aparece en ninguna bandeja
  • Queda en historial con estado final (Expedido/Eliminado)
  • Documentos permanecen en repositorio

9. Notas Importantes

  1. Comentario obligatorio en todos los escritorios (máx 512 chars)
  2. Campos cliente (nombres, doc) = solo lectura
  3. Ramo/Producto/Subproducto = solo lectura post-creación
  4. EstadoNeg = obligatorio y editable (usuario debe actualizarlo)
  5. Outsourcing (SI/NO) = obligatorio
  6. Devolución requiere Causa de Devolución
  7. Número póliza máx 13 dígitos
  8. Clave asesor = 5 dígitos (10000-99999)
  9. El botón determina el destino — NO hay lógica automática
  10. Escritorios paso único (CAV, Outsourcing, Gestión, Conocimiento) solo pueden completar
  11. Flujo Confidencial = mismos campos y reglas, solo acceso restringido
  12. Caso Externo no se puede modificar post-creación
  13. Servicio de Marcas se consulta automáticamente al abrir caso
  14. Si ContinuarProceso = "S" → campo "Actualizar Estado" se oculta
